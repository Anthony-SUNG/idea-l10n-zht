如果不想在每次修改 go.mod 後運行 go list ，請按 &shortcut:ShowSettings; 打開設置，並導航至 Build, Execution, Deployment | Build Tools 。清除 Reload project after changes in the build scripts 復選框。

默認情況下，go.mod 文件發生任何變化後 GoLand 都會重新加載項目。但是如果選擇 External changes 選項，則在 IDE 中編輯文件時 go list 不會自動執行。這種情況下，您將看到“加載 Go 模塊變更”圖標。完成編輯並點擊該圖標以加載變更。
