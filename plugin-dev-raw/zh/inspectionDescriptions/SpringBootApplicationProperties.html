報告 Spring Boot 應用程序 .properties 配置文件中未解析和棄用的配置鍵和無效值，這可能導致運行時錯誤。

示例：


server.port=invalid # 報告 '無法將 'invalid' 轉換為 java.lang.Integer'

如果已棄用的配置鍵有替換鍵，您可以應用 '使用替換鍵' 快速修復。

示例：


logging.path=${path} # 報告 '已棄用的配置屬性 'logging.path''

在應用快速修復后：


logging.file.path=${path}

如果 spring-configuration-metadata.json 中未定義配置鍵，您可以應用創建 META-INF/spring-additional-configuration-metadata.json 文件並定義必要鍵的 '定義配置鍵' 快速修復。

示例：


new.key=value #報告 '無法解析配置屬性 'new.key''

應用快速修復后，以下內容將添加到 META-INF/spring-additional-configuration-metadata.json：


{
  "properties": [
    {
      "name": "new.key",
      "type": "java.lang.String",
      "description": "Description for new.key."
  }
] }

該檢查還會高亮顯示列表和映射配置鍵中的索引表示法錯誤。

示例：


spring.datasource.schema[]=${schema} #報告 '缺少索引值'

使用替換令牌選項定義用于配置文件中值占位符的令牌。 這些令牌以 beginToken*endToken 的形式指定。 如果沒有 *，則假定開始和結束的令牌相同。

例如，開始和結束令牌的默認值都是 @，這使您可以定義占位符，例如 some.property=@another.property@。

替換令牌 (@property.key@) 內的值不會高亮顯示。
