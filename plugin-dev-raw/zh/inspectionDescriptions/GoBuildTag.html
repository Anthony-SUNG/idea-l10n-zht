報告構建標記的格式錯誤和放置在錯誤位置的構建標記。 go 工具要求將構建標記放置在特定位置並以特殊語法編寫。 如果不滿足這些要求，則可能會忽略構建標記或可能會意外地從構建中刪除文件。

請參閱 golang.org 上的 Build Constraints 頁面。

示例：

package main

// +build ignore

func main() {}

// +build ignore 部分應該在軟件包聲明之前。 要解決此問題，您可以應用在軟件包前面放置構建標籤快速修復。 在應用快速修復後：

// +build ignore

package main

import "fmt"
