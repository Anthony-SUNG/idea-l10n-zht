報告在屬性訪問器上重復使用非 @Repeatable 注解的情況。

由于多次使用非 @Repeatable 注解，這兩種注解的使用都會出現在字節碼中，從而導致反射調用中的歧義。

從 Kotlin 1.4 開始，必須將注解標記為 @Repeatable 或不重復注解，否則會導致編譯錯誤。

示例：


  annotation class Foo(val x: Int)

  @get:Foo(10)
  val a: String
      @Foo(20) get() = "foo" // 注解重復兩次，但未被標記為 @Repeatable

僅當項目或模塊的 Kotlin 語言級別為 1.4 或更高時，此檢查才會報告。
