報告接受 boolean 形參的 public 方法。

如果某個 public 方法（API 的一部分）不是 setter，則向該方法添加 boolean 形參几乎始終是一種不好的做法。 使用此類方法讀取代碼時，如果不查看源代碼或文檔，會很難理解 boolean 的含義。

此問題也被稱為布爾陷阱。 boolean 形參通常可以替換為 enum。

示例：


  // 警告：在查看對此方法的
  // 調用時，很難理解
  // 布爾形參的含義
  public boolean setPermission(File f,
                               int access,
                               boolean enable,
                               boolean ownerOnly) {
    // ...
  }

使用僅報告具有多個布爾形參的方法選項可以僅在一個方法包含多個布爾形參時发出警告。
