報告修飾符不符合規範首選順序（如 Java 語言規範中所述）的聲明。

示例：


  class Foo {
    native public final void foo();
  }

在應用快速修復後：


  class Foo {
    public final native void foo();
  }

使用檢查設置可以：

  • 切換位置錯誤的注解的報告： （帶有 ElementType.TYPE_USE 的不緊接在類型之前和修飾符關鍵字之後的注解，或其他不在修飾符關鍵字之前的注解）。 禁用此選項後，任何注解都可以放置在修飾符關鍵字之前或之後。 將始終報告在修飾符關鍵字之間帶有注解的修飾符列表。

  • 指定 ElementType.TYPE_USE 注解是否應直接置於類型之前，即使該注解指定了其他目標。
