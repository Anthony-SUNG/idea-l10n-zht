報告與可 null 性注解相關的問題。

示例：

  • 重寫方法未注解： 
    
abstract class A {
  @NotNull abstract String m();
}
class B extends A {
    String m() { return "empty string"; }
}
  • 注解的基元類型：@NotNull int myFoo;
  • @Nullable 和 @NotNull 位于同一成員上：@Nullable @NotNull String myFooString;
  • 可 null 元素的集合被分配到非 null 元素的集合中： 
    
void testList(List<@Nullable String> nullableList) {
   List<@NotNull String> list2 = nullableList;
}

使用配置注解按鈕可指定可 null 性注解，使用復選框可細調該檢查提供警告的位置。

僅當項目或模塊的語言級別為 5 或更高，並且類路徑上提供可 null 性注解時，此檢查才會報告。
