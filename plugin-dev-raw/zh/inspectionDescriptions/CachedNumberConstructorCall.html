報告具有基元 long、integer、short 或 byte 實參的新 Long、Integer、Short 或 Byte 對象的實例化。

建議使用 Java 5 中引入的 static 方法 function valueOf() { [native code] }() 預設情況下，此方法緩存值介於 -128 到 127（包含）之間的對象。

示例：


  Integer i = new Integer(1);
  Long l = new Long(1L);

應用快速修復後，程式碼變為：


  Integer i = Integer.function valueOf() { [native code] }(1);
  Long l = Long.function valueOf() { [native code] }(1L);

僅當項目或模塊的語言級別為 5 或更高級別時，此檢查才會報告。

使用忽略具有字串實參的新數字表達式選項可以忽略對包含 String 實參的數字構造函數的調用。

使用僅在構造函數為 @Deprecated 時報告選項，僅報告對棄用的構造函數的調用。 Long、Integer、Short 和 Byte 構造函數自 JDK 9 起已被棄用。
