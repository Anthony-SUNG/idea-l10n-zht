報告與周圍類的字段同名的方法形參。 由于進行此類命名，在有意使用同名字段的情況下，您可能會誤用該形參。

建議通過快速修復重命名該形參。

示例：


  class Main {
    private String value;

    public Main(String value) {
      value = value.toUpperCase();
    }
  }

您可以為此檢查配置以下選項：

  1. 對屬性 setter 忽略 - 忽略簡單 setter 的形參。
  2. 忽略從子類不可見的超類字段 - 忽略超類中對此方法不可見的 private 字段。
  3. 對構造函數忽略 - 忽略構造函數的形參。
  4. 對 abstract 方法忽略 - 忽略 abstract 方法的形參。
  5. 對隱藏實例字段的 static 方法形參忽略 - 忽略隱藏實例字段的 static 方法的形參，並忽略隱藏外部類實例字段的 static 內部類中實例方法的形參。 雖然不是嚴格意義上的隱藏，但此類形參仍然令人困惑。
