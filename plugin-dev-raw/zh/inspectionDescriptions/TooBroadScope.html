報告任何可以移動到更小範圍的變量聲明。

這種檢查對於方法開頭的 Pascal 樣式聲明特別有用。 此外，範圍太廣的變量在重構後也經常保留。

sb 移動的示例：


  StringBuilder sb = new StringBuilder();
  System.out.println();
  sb.append(1);

在應用快速修復後：


  System.out.println();
  StringBuilder sb = new StringBuilder();
  sb.append(1);

配置檢查：

  • 使用僅報告可移入內部塊的變量選項可只報告那些可以移動到更深代碼塊中的變量。 例如，當啟用該選項時，將建議對 sb 變量進行移動，但不建議對以下代碼進行移動： 
    
  StringBuilder sb = new StringBuilder(a);
  if (flag) {
    sb.append(1);
  }
  • 使用報告使用新表達式作為初始值設定項的變量(可能不安全) 選項可以報告使用新表達式初始化的變量。 這使得當構造函數具有非本地副作用時，檢查可能會不安全。 例如，啟用該選項時，將建議對 foo 變量進行移動： 
    
  class Foo {
    static List<Foo> fooList = new ArrayList<>();
    String bar;

    Foo(String bar) {
        this.bar = bar;
        fooList.add(this);
    }

    public static void main(String[] args) {
      Foo foo = new Foo("bar"); // 雖不安全，但也可以移動
      System.out.println(fooList.size());
      System.out.println(foo.bar);
    }
  }
