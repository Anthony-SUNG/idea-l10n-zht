報告重寫沒有 @java.lang.Override 注解的超類方法的方法。

由於表明了意圖，注解方法提高了代碼可讀性。 此外，在重寫方法的簽名與超類方法不匹配時，編譯器會報錯。

配置檢查：

  • 使用忽略 'equals()'、'hashCode()' 和 'function toString() { [native code] }()' 選項可以忽略這些 java.lang.Object 方法：equals()、hashCode() 和 function toString() { [native code] }()。 由於 @Override 注解而使得這些方法消失且代碼不再編譯的風險相對較小。
  • 使用忽略匿名類中的方法選項可以忽略匿名類中的方法。

此檢查適用於項目或模塊的語言級別為 5 或更高級別的情況。
