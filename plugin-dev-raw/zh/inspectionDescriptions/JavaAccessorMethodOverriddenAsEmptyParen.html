報告具有重寫 Java 訪問器方法的空形參子句的方法。

符合訪問器 JavaBean 命名協定的方法預計不會有副作用。 推薦的慣例是，在沒有形參且方法沒有副作用時使用無形參方法。 該慣例促進了統一訪問原則，即客戶端代碼不應受到將特性實現為字段或方法的決定的影響。

問題是 Java 沒有實現統一訪問原則，不允許沒有形參子句的方法。 為了填補這一空白，Scala 允許您使用沒有形參子句的方法來重寫帶有空形參子句的方法。

該快速修復會移除空形參子句。

示例：


  class TestException extends Exception {
    override def getMessage(): String = ???
  }

在應用快速修復後：


  class TestException extends Exception {
    override def getMessage: String = ???
  }
