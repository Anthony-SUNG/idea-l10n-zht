報告帶有未被保護條件包圍的非常量實參的日志記錄調用。 對日志記錄調用的實參的評估可能開銷巨大。 當日志記錄語句使用的級別禁用日志記錄時，使用臨界子句包圍日志記錄調用可避免這種開銷。 這對于最不嚴重的日志記錄調用級別（跟踪、調試、最精細）特別有用，因為這些調用在生產環境中最常被禁用。

示例：


  public class Principal {
    void bad(Object object) {
      if (true) {
        LOG.debug("log log log " + expensiveCalculation(object));
      }
      LOG.debug("some more logging " + expensiveCalculation(1));
    }

    void good(Object) {
      if (LOG.isDebug()) {
        LOG.debug("value: " + expensiveCalculation(object));
      }
    }
  }

配置檢查：

  • 使用記錄器類名字段可以指定使用的記錄器類名。

  • 使用該表可以指定此檢查應警告的日志記錄方法，以及相應的日志條件文本。

  • 使用標記所有不受保護的日志調用選項可讓檢查標記所有不受保護的日志調用，而不僅僅是那些具有非常量實參的調用。
