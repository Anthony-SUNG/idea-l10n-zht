報告類型形參被忽略的泛型類。 這種對泛型類型的原始使用在 Java 中雖然有效，但它違背了類型形參的用途，並且可能掩蓋錯誤。 此檢查反映出 javac 的 rawtypes 警告。

示例：


//警告：形參化類 'List' 的原始使用
List list = new ArrayList<String>();
//創建了字符串列表，但也接受整數
list.add(1);


//無警告，因為在數組創建期間不可能提供類型實參
IntFunction<List<?>[]> fun = List[]::new;

配置檢查：

  • 使用忽略新對象的構造選項，可忽略對象構造中使用的原始類型。
  • 使用忽略類型轉換選項，可忽略類型轉換中使用的原始類型。
  • 使用忽略類型形參無法編譯的位置選項，可忽略類型形參編譯失敗的情況（例如創建數組或重寫庫方法的情況）。
  • 使用忽略重寫方法的形參類型選項，可忽略重寫方法的形參中使用的類型形參。
  • 使用在自動快速修復不可用時忽略選項，可忽略快速修復不可用的情況。
僅當項目或模塊的語言級別為 5 或更高時，此檢查才會報告。
