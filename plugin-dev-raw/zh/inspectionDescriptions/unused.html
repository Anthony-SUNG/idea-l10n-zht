報告指定檢查範圍中未使用或無法從入口點到達的類、方法或字段。

入口點可以是主方法、測試、指定範圍外提及的類、可從 module-info.java 訪問的類，等等。 還可以使用名稱模式或注解來配置自定義入口點。

示例：


  public class Department {
    private Organization myOrganization;
  }

在本例中，Department 顯式引用 Organization，但如果 Department 類本身未使用，那麼該檢查將報告這兩個類。

該檢查還會報告其方法、所有方法實現和重寫器未使用的形參，以及已聲明但未使用的局部變量。

注意：在編輯器中高亮顯示代碼期間可能不會報告某些未使用的成員。 由於性能原因，僅當非 private 成員的名稱很少出現在項目中時，才會對其進行檢查。 要查看所有結果，請從主菜單中選擇代碼 | 檢查代碼或代碼 | 分析代碼 | 按名稱運行檢查 來運行檢查。

使用下面的可見性設置可以配置要報告的成員。 例如，配置報告 private 方法只表明將報告 private 內部類的 public 方法，頂層類的 protected 方法將被忽略。

使用入口點頁籤可以配置檢查運行期間要考慮的入口點。

在檢查結果準備就緒後，可以手動添加入口點。

如果您的代碼使用不受支持的框架，可以選擇以下幾個選項：

  • 如果框架依賴於注解，使用注解…按鈕可配置框架注解。
  • 如果框架不依賴於注解，可嘗試配置框架應有的類名模式。

這樣，框架內部項可訪問的注解代碼將被視為已使用。
