報告在方法簽名中聲明但方法本身或其實現和派生從未拋出的異常。

該檢查會忽略與序列化相關的方法，例如方法 readObject() 或 writeObject()。

示例：


  void method() throws InterruptedException {
    System.out.println();
  }

該快速修復會移除聲明中不必要的異常，並規范化冗余的 try-catch 語句：


  void method() {
    System.out.println();
  }

注意：在編輯器中高亮顯示時可能不會報告某些異常。 由于性能原因，僅當非 private 方法的名稱很少出現在項目中時，才會對其進行檢查。 要查看所有結果，請從主菜單中選擇代碼 | 檢查代碼或代碼 | 分析代碼 | 按名稱運行檢查來運行檢查。

使用忽略由入口點方法拋出的異常選項可以忽略由入口點方法拋出的異常。 入口點方法在以下檢查中配置：編輯器 | 檢查 | Java | 聲明冗余 | 未使用的聲明。
