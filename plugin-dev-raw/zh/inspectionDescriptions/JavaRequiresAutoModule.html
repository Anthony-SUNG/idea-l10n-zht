報告 requires 指令中自動模塊的用法。

自動模塊不可靠，因為它可能依賴于類路徑中的類型，並且當它轉換為顯式模塊時，它的名稱和導出的軟件包可能會改變。

對應于 -Xlint:requires-automatic 和 -Xlint:requires-transitive-automatic Javac 選項。 第一個選項增加了對何時使用自動模塊的了解。 第二個選項則是警示模塊的作者，建立自動模塊的隱含可讀性將使該模塊的用戶面臨風險。

示例：


  //module-info.java
  module org.printer {
    requires transitive drivers.corp.org; // 在 'drivers.corp.org' 為自動模塊時報告
  }

使用僅高亮顯示傳遞依賴選項，可僅對傳遞依賴发出警告。
